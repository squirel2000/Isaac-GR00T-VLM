## 背景——塑造本計劃的解答（完整細節見聊天記錄）

- **Q（可行性）：Cosmos-R2 能在 4090 上進行 train+merge+swap 嗎？** 可以。Cosmos-Reason2-2B 約 2B 參數（bf16 約 4–5 GB）；LoRA + grad-checkpointing + batch=1 可舒適裝入 24 GB。`merge_lora` 和 `swap_backbone` 體積小／純 safetensors。只有全 VLA FFT 才需要 H100。
- **Q（提取）：Cosmos-R2 如何從 GR00T N1.7「切割出來」？** 透過 state_dict 鍵前綴。`swap_backbone.py` 僅以鍵/形狀交集替換 `backbone.model.*`（即 `Qwen3VLForConditionalGeneration` VLM）；`action_head.*`（vlln + 4 層 `vl_self_attention` adapter + 32 層 DiT + encoders/decoder）保持不動。
- **Q：4 層 adapter 是 VLM backbone 的一部分嗎？** 不是。`vl_self_attention`（`num_layers=4`）在 `Gr00tN1d7ActionHead`（`gr00t_n1d7.py:90`）內部實例化，前綴為 `action_head.vl_self_attention.*`，屬於 action head，**不會被 swap**。
- **Q：VLM 還是 LLM？** Cosmos-R2 是 **VLM**（`.visual` ViT 24L + `.language_model` Qwen3 28L，隱藏維度 2048）。在 VLA 內部只使用 `hidden_states[-1]`（LM head 不使用）；合併後的 Product A 是完整的獨立 VLM。
- **Q：select_layer 12 還是 16？** 程式碼*預設*為 12（`configs/model/gr00t_n1d7.py:47`）；此處所有已部署的 `N1_7_fft_*` checkpoint 均使用 **16**。VLA 將 28 層 LLM 截斷至前 `select_layer` 層（`qwen3_backbone.py:87-88`）。
- **Q：`train_vlm_lora.py` 是標準 transformers 流程嗎？** 是的——`AutoProcessor` + `Qwen3VLForConditionalGeneration` + `peft.get_peft_model` + `Trainer`/`TrainingArguments` 是標準 HF VLM SFT 配方。但它**專屬於 Qwen3-VL 架構**（類別名稱與 `lora_target="llm"` 的 proj 名稱均假設為 Qwen3 層）；若使用不同的 VLM 家族，需要自己的 `AutoModelForImageTextToText` 子類別與 target 模組。這與 GR00T N1.7 訓練不同——後者使用 GR00T 自家 trainer 訓練**整個 VLA**（backbone+adapter+DiT，flow-matching action loss），而非 HF `Trainer`，也非 CausalLM loss。

```python
# Six distinct manipulation phases of the can-sort episode, in temporal order.
_PHASES = ("approach", "grasp", "pick", "move_to_plate", "place", "retract")
_PHASE_FRACS = {
    "approach": 0.12, "grasp": 0.30, "pick": 0.45,
    "move_to_plate": 0.62, "place": 0.80, "retract": 0.96,
}
_PHASE_ORDER = {p: i for i, p in enumerate(_PHASES)}


def _sample_phase_frames(length: int, phases: tuple[str, ...] = _PHASES) -> list[tuple[int, str]]:
    """Map each named manipulation phase to one representative frame index."""
    out = []
    for ph in phases:
        frac = _PHASE_FRACS.get(ph, 0.5)
        idx = min(length - 1, max(0, int(frac * (length - 1))))
        out.append((idx, ph))
    return out
```
