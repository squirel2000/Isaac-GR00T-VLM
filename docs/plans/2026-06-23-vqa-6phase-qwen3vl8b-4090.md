## Background — answers that shape this plan (see chat for full detail)

- **Q (feasibility): Can Cosmos-R2 train+merge+swap on the 4090?** Yes. Cosmos-Reason2-2B is ~2B params (~4–5 GB bf16); LoRA + grad-checkpointing + batch=1 fits 24 GB comfortably. `merge_lora` and `swap_backbone` are tiny / pure-safetensors. Only full-VLA FFT needs H100.
- **Q (extraction): how is Cosmos-R2 "cut out" of GR00T N1.7?** By state_dict key prefix. `swap_backbone.py` replaces only `backbone.model.*` (= the `Qwen3VLForConditionalGeneration` VLM) by key/shape intersection; `action_head.*` (vlln + 4-layer `vl_self_attention` adapter + 32-layer DiT + encoders/decoder) is untouched.
- **Q: is the 4-layer adapter part of the VLM backbone?** No. `vl_self_attention` (`num_layers=4`) is instantiated inside `Gr00tN1d7ActionHead` (`gr00t_n1d7.py:90`) → prefix `action_head.vl_self_attention.*`. It belongs to the action head and is **not** swapped.
- **Q: VLM or LLM?** Cosmos-R2 is a **VLM** (`.visual` ViT 24L + `.language_model` Qwen3 28L, hidden 2048). Inside the VLA only `hidden_states[-1]` is used (LM head unused); the merged Product A is a full standalone VLM.
- **Q: select_layer 12 vs 16?** Code *default* is 12 (`configs/model/gr00t_n1d7.py:47`); every deployed `N1_7_fft_*` checkpoint here uses **16**. The VLA truncates the 28-layer LLM to the first `select_layer` layers (`qwen3_backbone.py:87-88`).
- **Q: is `train_vlm_lora.py` the standard transformers flow?** Yes — `AutoProcessor` + `Qwen3VLForConditionalGeneration` + `peft.get_peft_model` + `Trainer`/`TrainingArguments` is the canonical HF VLM SFT recipe. It is **architecture-specific to Qwen3-VL** (the class name and `lora_target="llm"` proj names assume Qwen3 layers); a different VLM family needs its own `AutoModelForImageTextToText` subclass + target modules. It differs from GR00T N1.7 training, which uses GR00T's own trainer over the **whole VLA** (backbone+adapter+DiT, flow-matching action loss) — not HF `Trainer`, not a CausalLM loss.

```python
# Six distinct manipulation phases of the can-sort episode, in temporal order.
_PHASES = ("approach", "grasp", "pick", "move_to_plate", "place", "retract")
_PHASE_FRACS = {
    "approach": 0.12, "grasp": 0.30, "pick": 0.45,
    "move_to_plate": 0.62, "place": 0.80, "retract": 0.96,
}
_PHASE_ORDER = {p: i for i, p in enumerate(_PHASES)}


def _sample_phase_frames(length: int, phases: tuple[str, ...] = _PHASES) -> list[tuple[int, str]]:
    """Map each named manipulation phase to one representative frame index."""
    out = []
    for ph in phases:
        frac = _PHASE_FRACS.get(ph, 0.5)
        idx = min(length - 1, max(0, int(frac * (length - 1))))
        out.append((idx, ph))
    return out
```
